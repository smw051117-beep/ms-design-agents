**파일**: `docs/designs/20260526_일정관리에이전트.md`
**날짜**: 2026-05-26
**환경**: M365 Business Premium (외부망 단일 테넌트)

---

## [Orchestrator] 요구사항 분해 및 설계 방향

**핵심 확인 사항**:
- 금융권 망분리 **미적용** — M365 Business Premium 단일 테넌트
- 모든 처리가 동일 테넌트 내 M365 서비스로 완결
- Copilot Studio가 프론트엔드, Power Automate가 백엔드 역할

**4개 플로우 분해**:

| Flow | 역할 |
|------|------|
| Flow 1 | 일정 정보 수집 → Graph API 임원 조회 → 임원 선택 카드 |
| Flow 2 | 임원/부서장 알림 발송 (카드 유형 분기) |
| Flow 3 | 부서장 응답 수신 → 참석/미참석 캘린더 등록 |
| Flow 4 | 재참석 변경 처리 |

## [Architect] 시스템 아키텍처

### 컴포넌트 흐름도

```
본인 (서울업무팀)
    │ "일정 등록해줘"
    ▼
[Copilot Studio 에이전트]
    │ 일정정보 수집 완료 → HTTP POST
    ▼
[Flow 1: 일정등록_요청처리]
    │
    ├─ Graph API ──→ Entra ID [임원진 그룹] → 임원 목록
    │
    └─ Teams 적응형 카드 (Card 1: 임원 선택 체크박스) ──→ 본인
              │ 임원 선택 완료
              ▼
[Flow 2: 알림_발송]
    │
    ├─ For each 임원:
    │     ├─ Graph API → [임원명_부서장 그룹] → 부서장 목록
    │     ├─ Teams 카드 (Card 3: 열람 전용) ──→ 팀장님(임원)
    │     └─ Teams 카드 (Card 2: 참석/미참석) ──→ 팀원1(부서장)
    │                       │ 버튼 클릭
    │                       ▼
    │             [Flow 3: 응답처리_캘린더등록]
    │                       │
    │             ┌─────────┴──────────┐
    │          참석                미참석
    │             │                    │
    │    캘린더 3개 등록        캘린더 3개 "미참석" 등록
    │    Teams 알림 ✅          Teams 알림 ❌
    │
    └─ (재참석 변경 시)
[Flow 4: 재참석_변경] ─→ 캘린더 3개 수정 → Teams 알림 🔄
```

### Entra ID 그룹 구조

```
임원진 (그룹)
└── 팀장님

팀장님_부서장 (그룹)
└── 팀원1
```

> **핵심 설계 원칙**: 임원 추가/삭제 시 Entra ID 그룹만 수정 → 플로우 자동 반영

### 망 배치 결정

- **단일 테넌트 (M365 Business Premium 외부망 테넌트)**
- 모든 컴포넌트가 동일 테넌트 내 처리
- 금융권 망분리 규정 미적용 (일반 기업 환경)
- Graph API는 동일 테넌트 내 서비스 프린시펄로 인증

## [Developer] 상세 설계

### 1. Entra ID 그룹 구성

| 그룹 이름 | 멤버 | 용도 |
|----------|------|------|
| `임원진` | 팀장님 | 임원 목록 조회 |
| `팀장님_부서장` | 팀원1 | 팀장님 하위 부서장 조회 |

> 임원 추가 시: `{신규임원명}_부서장` 그룹 추가 생성

---

### 2. Power Automate 플로우 설계

#### Flow 1: `일정등록_요청처리`

```
트리거: HTTP 요청 (Copilot Studio → Power Automate)
Body:
  {
    "일정제목": "string",
    "시작시간": "2026-05-30T10:00:00",
    "종료시간": "2026-05-30T11:00:00",
    "장소": "string",
    "요청자UPN": "user@tenant.com"
  }

Step 1. [HTTP] Graph API 호출
  GET https://graph.microsoft.com/v1.0/groups/{임원진그룹ID}/members
  → 임원 목록 배열 파싱

Step 2. [Teams] 적응형 카드 발송 (Card 1)
  → 채널: 본인 Teams 채팅
  → Wait for adaptive card response (응답 대기)

Step 3. [응답 파싱] 선택된 임원 UPN 배열 추출

Step 4. [Flow 2 호출] HTTP POST
  → 일정정보 + 선택된임원목록 전달
```

**필요 Graph API 권한**:
- `GroupMember.Read.All`
- `User.Read.All`

---

#### Flow 2: `알림_발송`

```
트리거: HTTP 요청 (Flow 1에서 호출)
Input:
  {
    "일정정보": {...},
    "선택된임원목록": ["upn1@...", "upn2@..."],
    "요청자UPN": "string"
  }

[Apply to each 임원 in 선택된임원목록]
  │
  Step 1. [HTTP] 부서장 그룹 조회
    GET /groups/{임원명_부서장그룹ID}/members
    → 부서장 UPN 목록 추출
  │
  Step 2. [Teams] 임원에게 Card 3 (열람 전용) 발송
    수신자: 임원 UPN
    메시지: "아래 일정이 등록될 예정입니다.
             참석 여부는 담당 부서장이 결정합니다."
    버튼: 없음
  │
  [Apply to each 부서장]
    Step 3. [Teams] 부서장에게 Card 2 (응답 필요) 발송
      수신자: 부서장 UPN
      메시지: "아래 일정에 대한 참석 여부를 선택해 주세요."
      버튼: [✅ 참석] [❌ 미참석]
      → Wait for adaptive card response
      │
      Step 4. [Flow 3 호출]
```

---

#### Flow 3: `응답처리_캘린더등록`

```
트리거: HTTP 요청 (Flow 2에서 부서장 응답 후 호출)
Input:
  {
    "응답": "참석" | "미참석",
    "부서장UPN": "string",
    "부서장이름": "string",
    "임원UPN": "string",
    "요청자UPN": "string",
    "일정정보": {...}
  }

[Condition: 응답 == "참석"]

─── YES (참석) ───────────────────────────────────────
  [Office 365 Outlook] 이벤트 만들기 × 3
    ① 부서장 캘린더: "{일정제목}"
    ② 임원 캘린더:   "{일정제목}"
    ③ 본인 캘린더:   "{일정제목}"

  [Teams] 본인 채널 알림
    "✅ [부서장이름]이 [일정제목]에 참석 확정했습니다."

─── NO (미참석) ──────────────────────────────────────
  [Office 365 Outlook] 이벤트 만들기 × 3
    ① 부서장 캘린더: "{일정제목} - 미참석"
    ② 임원 캘린더:   "{일정제목} - {부서장이름} 미참석"
    ③ 본인 캘린더:   "{일정제목} - {부서장이름} 미참석"

  [Teams] 본인 채널 알림
    "❌ [부서장이름]이 [일정제목]에 미참석 의사를 밝혔습니다."
```

**Outlook 커넥터 액션**: `Office 365 Outlook - 이벤트 만들기 (V4)`  
또는 Graph API `POST /users/{userId}/events`

---

#### Flow 4: `재참석_변경처리`

```
트리거: HTTP 요청 (Copilot Studio에서 호출)
Input:
  {
    "부서장UPN": "string",
    "부서장이름": "string",
    "임원UPN": "string",
    "일정제목": "string",
    "요청자UPN": "string"
  }

Step 1. [Office 365 Outlook] 이벤트 검색
  → 3개 캘린더에서 "미참석" 포함 일정 검색

Step 2. [Office 365 Outlook] 이벤트 업데이트 × 3
  ① 부서장: "{일정제목} - 미참석" → "{일정제목}"
  ② 임원:   "{일정제목} - {부서장이름} 미참석" → "{일정제목}"
  ③ 본인:   "{일정제목} - {부서장이름} 미참석" → "{일정제목}"

Step 3. [Teams] 본인 채널 알림
  "🔄 [부서장이름]이 [일정제목]에 재참석으로 변경했습니다."
```

### 3. Copilot Studio 에이전트 토픽 설계

#### Topic 1: 일정 등록 요청

| 항목 | 내용 |
|------|------|
| 트리거 문구 | "일정 등록", "일정 추가", "일정 잡아줘", "미팅 등록" |
| 수집 변수 | 일정제목, 시작날짜시간, 종료날짜시간, 장소 |

```
[Question] 일정 제목을 알려주세요.
  → 저장: Var_일정제목

[Question] 시작 날짜와 시간을 알려주세요. (예: 5월 30일 오전 10시)
  → 저장: Var_시작시간

[Question] 종료 시간을 알려주세요.
  → 저장: Var_종료시간

[Question] 장소를 알려주세요.
  → 저장: Var_장소

※ 미입력 2개 이상: 한 번에 묶어서 질문
  예) "일정 제목과 장소를 알려주세요."

[Confirm 메시지]
  "아래 일정 정보가 맞나요?
   📌 제목: {Var_일정제목}
   🕐 시작: {Var_시작시간}
   🕑 종료: {Var_종료시간}
   📍 장소: {Var_장소}
   [확인] [수정]"

[Action] Flow 1 호출 (Power Automate HTTP 액션)
  → "일정 등록 요청이 접수되었습니다. 잠시 후 임원 선택 카드가 전송됩니다."
```

#### Topic 2: 재참석 변경

| 항목 | 내용 |
|------|------|
| 트리거 문구 | "재참석", "참석으로 변경", "다시 참석할게요" |
| 수집 변수 | 부서장이름, 일정제목 |

```
[Question] 어떤 분의 재참석을 변경할까요?
  → 저장: Var_부서장이름

[Question] 어떤 일정인가요?
  → 저장: Var_일정제목

[Confirm 메시지]
  "[Var_부서장이름]의 [Var_일정제목]을 재참석으로 변경할까요?
   [확인] [취소]"

[Action] Flow 4 호출
  → "재참석으로 변경되었습니다."
```

## [Security] 보안 검토

**결론: 설계 승인 (조건 4개 충족 필요)**

| # | 검토 항목 | 결과 | 조치 |
